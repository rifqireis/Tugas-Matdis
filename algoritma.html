01 — Lokal O(V + E)

Degree Centrality

Algoritma paling sederhana: hitung jumlah tetangga setiap node, normalisasi dengan (N−1). Jawaban langsung untuk pertanyaan siapa yang paling banyak berkolaborasi?

Definisi formal
CD(v) = deg(v) / (N − 1)

Untuk graf 28 node, pembagi-nya 27. Node yang terhubung dengan semua orang akan mendapat skor 1.0.

Visual

Node tengah: deg = 5 → skor 5/6 ≈ 0.83

Pseudocode
function degreeCentrality(G):
  N = |V(G)|
  scores = {}
  for v in V(G):
    scores[v] = |neighbors(v)| / (N − 1)
  return scores

// Kompleksitas: O(V + E) — sekali jalan, sangat efisien
// Memori:       O(V)
Kelemahan — tidak melihat kualitas tetangga. Akun dengan 50 follower kecil bisa "menang" dari akun dengan 5 follower yang semuanya jutaan.
02 — Iteratif global O(k · (V+E))

PageRank

Algoritma dari Larry Page (Google, 1998). Mengasumsikan pengaruh "mengalir" melalui edge. Node berpengaruh adalah node yang ditunjuk oleh node-node berpengaruh lain — rekursif.

Formula
PR(v) = (1 − d) / N + d · Σu ∈ in(v) [ PR(u) / |out(u)| ]
d
0.85
damping factor
Iterasi
60
iterasi (fixed)
Tol
10−6
|ΔPR| threshold
Langkah-langkah
01
Inisialisasi

Setiap node v diberi PR awal = 1/N. Untuk graf 28 node, PR awal ≈ 0.0357 untuk semua.

02
Distribusi pengaruh

Setiap node membagi PR-nya secara merata ke semua tetangga. Tetangga menerima "voting" — semakin banyak vote dari node penting, semakin tinggi PR-nya.

03
Damping factor (d = 0.85)

Mensimulasikan probabilitas user random meninggalkan jaringan (15%) — mencegah skor terjebak di "sink node". Tanpa damping, algoritma bisa divergen.

04
Iterasi sampai konvergen

Ulangi langkah 2–3 sampai perubahan PR antar iterasi sangat kecil. Pada implementasi kami, berjalan 60 iterasi fixed — cukup untuk dataset 28 node.

Pseudocode
function pageRank(G, d=0.85, maxIter=100, tol=1e-6):
  N = |V(G)|
  PR = { v: 1/N for v in V(G) }

  for k in 1..maxIter:
    PR_new = { v: (1 − d) / N for v in V(G) }
    for u in V(G):
      if |out(u)| > 0:
        share = d · PR[u] / |out(u)|
        for v in out(u):
          PR_new[v] += share
    if max(|PR_new[v] − PR[v]| for v) < tol:
      break
    PR = PR_new
  return PR
Pada dataset kami — @willie27_ mendapat PR 3.42% — di atas median (1.2%) karena terhubung ke 3 hub besar (Baim Wong, Jess No Limit) plus banyak kreator Bengkulu.
03 — Iteratif + bobot zona O(k · (V+E))

Weighted PageRank

Ekstensi PageRank standar: setiap edge diberi bobot berdasarkan community zone asal dan tujuan. Kreator lokal Bengkulu mendapat amplifikasi ×1.5 saat berkolaborasi sesama lokal, sehingga pengaruh intra-komunitas tidak tenggelam di bawah dominasi kreator nasional.

Formula
PRw(v) = (1 − d) / N + d · Σu ∈ in(v) [ PRw(u) · wnorm(u, v) ]
wnorm(u, v) = communityFactor(u.com, v.com) / Σu→x communityFactor(u.com, x.com)
Community factor
Source → Target Faktor Alasan
lokal → lokal ×1.5 Perkuat ikatan intra-komunitas lokal
* → lokal ×1.3 Boost koneksi masuk ke komunitas lokal
lokal → nasional ×0.9 Sedikit redam aliran keluar dari lokal
selainnya ×1.0 Netral
Langkah-langkah
01
Hitung bobot tiap edge

Untuk setiap edge (u→v), tentukan communityFactor(u.com, v.com). Hasilnya adalah bobot mentah.

02
Normalisasi per-node

Total bobot outgoing setiap node dinormalisasi ke 1.0, sehingga total distribusi rank tetap 100%.

03
Iterasi PageRank berbobot

Sama seperti PageRank standar, tapi setiap distribusi rank menggunakan bobot ternormalisasi wnorm(u,v).

04
80 iterasi (lebih banyak dari standar)

Bobot tidak simetris membuat konvergensi sedikit lebih lambat. Dataset kami stabil setelah 80 iterasi.

Pseudocode
function weightedPageRank(G, d=0.85, iters=80):
  // Hitung bobot tiap edge
  for (u→v) in edges(G):
    w[u][v] = communityFactor(u.com, v.com)

  // Normalisasi per-node
  for u in V(G):
    total = Σ w[u][x] for x in out(u)
    if total > 0: w[u][x] /= total

  // Iterasi PageRank
  N = |V(G)|; PR = { v: 1/N }
  for k in 1..iters:
    PR_new = { v: (1-d)/N }
    for (u→v) in edges(G):
      PR_new[v] += d * PR[u] * w[u][v]
    PR = PR_new
  return PR
Efek pada dataset — Kreator lokal yang banyak berkolaborasi sesama lokal mendapat PageRank lebih tinggi dibanding rata-rata. @mariorioio__ naik signifikan karena 11 kreator lokal menunjuknya — masing-masing dengan bobot lokal→lokal ×1.5.
04 — Global O(V · E)

Betweenness Centrality

Menghitung berapa banyak shortest path antar pasangan node yang melewati node v. Tinggi = node ini adalah jembatan kritis dalam komunikasi jaringan.

Definisi
CB(v) = Σs≠v≠t [ σst(v) / σst ]

σst = total shortest path dari s ke t; σst(v) = jumlah yang melewati v. Implementasi: algoritma Brandes (2001) — BFS dari setiap source.

Pada dataset — Node dengan betweenness tinggi adalah "jembatan" antar komunitas. @willie27_ dan @mariorioio__ punya betweenness terbesar. Putus mereka, dan komunikasi antar klaster terganggu.
05 — Komparasi

Memilih algoritma yang tepat.

Algoritma Pertanyaan Kompleksitas Skala
Degree Siapa paling banyak kolaborasi? O(V + E) Lokal
PageRank Siapa paling berpengaruh secara kualitas? O(k · (V+E)) Global iteratif
Weighted PR Siapa berpengaruh dalam konteks komunitasnya? O(k · (V+E)) Global + bobot zona
Betweenness Siapa jembatan komunikasi? O(V · E) Global, mahal

Referensi: Brandes, U. (2001). A faster algorithm for betweenness centrality. Page, L. & Brin, S. (1998). The anatomy of a large-scale hypertextual web search engine. Xing, W. & Ghorbani, A. (2004). Weighted PageRank algorithm.
