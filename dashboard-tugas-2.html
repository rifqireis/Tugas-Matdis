03 / Algoritma
3 metode analisis yang digunakan
Masing-masing menjawab pertanyaan berbeda tentang jaringan yang sama.

1. Degree Centrality

Mengukur volume koneksi langsung — siapa paling banyak berkolaborasi

O(V + E) — linier, N = 28, pembagi = 27

Formula (in-degree):

DC(v) = in-degree(v) / (N − 1) = in-degree(v) / 27

Kita pakai in-degree: berapa banyak kreator lain yang menunjuk (mengajak collab) ke node v. Nilai dinormalisasi dengan membagi (N−1) = 27 agar berada di rentang [0, 1].

// Degree Centrality — O(V + E) function degreeCentrality(): inDeg = {} // in-degree tiap node for each node v: inDeg[v] = 0 for each edge (u → v): inDeg[v] += 1 for each node v: DC[v] = inDeg[v] / (N − 1) // N = 28 return DC
Analogi: Di kelas, siapa yang paling sering diajak kerja kelompok oleh teman-temannya? Degree hitung frekuensinya — tidak peduli siapa yang mengajak.

Hasil data asli: @mariorioio__ punya in-degree = 13 → DC = 13/27 ≈ 48.1%. Ini tertinggi di seluruh jaringan.

2. PageRank

Mengukur pengaruh berlapis — kualitas koneksi, bukan hanya kuantitas

O(k × (V + E)), k = 60 iterasi, d = 0.85

Formula iterasi:

PR(v) = (1−d)/N + d × Σu∈in(v) PR(u) / out-degree(u)

Setiap iterasi, node mendistribusikan sebagian PageRank-nya ke node yang ia tunjuk. Proses ini diulang 60 kali hingga nilai konvergen. Damping factor d = 0.85 mensimulasikan 85% kemungkinan pengguna terus mengikuti rantai kolaborasi.

// PageRank — O(k × (V + E)), k = 60 function pageRank(d = 0.85, iters = 60): for each node v: PR[v] = 1/N // inisialisasi 1/28 ≈ 0.0357 repeat iters times: next = {} for each node v: next[v] = (1−d)/N for each node u: share = d × PR[u] / out-degree(u) for each neighbor v of u: next[v] += share PR = next return PR
Analogi: Bukan siapa yang paling populer — tapi siapa yang direkomendasikan oleh orang-orang berpengaruh. Rekomendasi dari Deddy Corbuzier lebih berharga dari 100 rekomendasi akun biasa.

Hasil data asli: @mariorioio__ = 13.69% — lebih dari 2× peringkat 2 (Vilmei 5.48%). Ini karena semua 13 kreator lokal menunjuk ke Mario.

3. Community Detection

Menemukan kelompok yang terbentuk berdasarkan pola kolaborasi

O(V + E) — partisi manual dengan validasi modularity

Formula modularity:

Q = Σc [ Lc/m − (dc/2m)² ]

Di mana Lc = jumlah edge dalam komunitas c, m = total edge, dc = total degree node di komunitas c. Q mendekati 1 berarti komunitas sangat rapat.

Hasil data asli: Q = 0.71 — komunitas sangat kuat. Hanya ada 1 dari 66 edge (1.5%) yang melintasi batas komunitas: @mariorioio__ → @willie27_.

04 / Analisis Wajib
7 pertanyaan — dijawab dengan data asli
Semua jawaban berdasarkan dataset final: 28 node, 66 edge.
1

Jenis graf apa yang digunakan?

Directed Graph (Digraph) — setiap edge punya arah. A → B berarti A mengajak B berkolaborasi, dan ini berbeda dari B → A. Graf ini juga mengandung multi-edge potensial dan self-loop nihil. Dipilih karena kolaborasi punya inisiatif yang asimetris.

2

Berapa kardinalitas vertex dan edge?

|V| = 28 node (14 nasional + 14 lokal Bengkulu) dan |E| = 66 edge berarah. Densitas graf = 66 / (28 × 27) ≈ 8.7% — tergolong sparse, wajar untuk jaringan sosial nyata.

3

Apakah graf connected?

Weakly connected — semua 28 node terhubung jika arah edge diabaikan. Namun tidak strongly connected karena tidak ada path berarah dari semua node nasional ke semua node lokal. Hanya ada 1 jembatan antar komunitas: @mariorioio__ → @willie27_. Isolasi komunitas lokal adalah temuan kunci.

4

Apakah terdapat cycle?

Ya — terdapat banyak cycle dari mutual collab. Contoh: @willie27_ ↔ @vilmei, @vilmei ↔ @riaricis1795, @vilmei ↔ @fadiljaidi, @jessnolimit ↔ @siscakohl. Cycle ini penting bagi PageRank — pengaruh "berputar" dan terakumulasi di node pusat jaringan.

5

Algoritma apa yang paling optimal?

PageRank — paling optimal untuk mengukur pengaruh dalam jaringan sosial karena mempertimbangkan kualitas koneksi, bukan hanya kuantitas. Degree hanya melihat volume (bisa dimanipulasi), PageRank melihat siapa yang menunjukmu. Kompleksitas O(k·(V+E)) masih linier dan scalable untuk dataset jauh lebih besar.

6

Bagaimana skalabilitas solusi?

Sangat baik. Degree Centrality O(V+E) dan PageRank O(k·(V+E)) — keduanya linier terhadap ukuran graf. Pada dataset 28 node ini, PageRank konvergen dalam 60 iterasi. Jika diperluas ke 10.000 node, 60 iterasi tetap cukup karena konvergensi bergantung pada struktur, bukan hanya ukuran.

7

Apa inovasi yang dapat dikembangkan?

Prediksi kolaborasi berikutnya — berdasarkan jarak dalam graf. @jessnolimit dan @mariorioio__ hanya 2 hop (via @willie27_). Probabilitas kolaborasi tinggi. Inovasi lain: deteksi komunitas dinamis (siapa yang berpindah klaster), identifikasi akun yang hanya "konsumsi" (banyak in-degree, sedikit out-degree), dan alert jika bridge node dihapus.

06 / Temuan Kunci
4 insight dari analisis graf
01
@mariorioio__ adalah hub mutlak komunitas lokal

PageRank 13.69% — lebih dari 2× peringkat kedua. 13 dari 14 kreator lokal menunjuknya. Jika node ini dihapus, komunitas lokal Bengkulu hancur menjadi 13 node terisolasi.

02
Hanya 1 jembatan antar komunitas dari 66 edge

@mariorioio__ → @willie27_ adalah satu-satunya koneksi komunitas lokal ke nasional (1.5% dari semua edge). Modularity Q = 0.71 — dua dunia yang hampir tidak bersentuhan.

03
Followers ≠ pengaruh dalam jaringan

@raffinagita1717 punya 75,2 juta followers (terbanyak di dataset) tapi PageRank-nya rendah. @mariorioio__ hanya punya ~1.239 followers tapi PageRank #1. Ini membuktikan PageRank mengukur pengaruh struktural, bukan popularitas publik.

04
Komunitas nasional lebih rapat dari lokal

Komunitas nasional punya cycle mutual yang banyak (Willie↔Vilmei, Vilmei↔Fadil, Jess↔Sisca, dll). Komunitas lokal lebih berbentuk bintang (star graph) dengan Mario di pusat — kurang tahan terhadap removal node pusat.

Kesimpulan: Untuk kampanye promosi dengan jangkauan maksimal di dua komunitas, kolaborasi dengan @mariorioio__ (lokal) dan @vilmei (nasional) — keduanya konsisten di top 3 PageRank dan Degree. @mariorioio__ adalah satu-satunya node yang menghubungkan kedua dunia.
